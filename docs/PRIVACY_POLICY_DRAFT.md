# River privacy policy — draft

> **Draft only — not legal advice or a published policy.** Obtain jurisdiction-specific legal review before relying on this document.

River processes account details, conversation content, approved memories, and limited operational metadata to provide a memory-aware companion. It does not intentionally store raw voice recordings. A memory is saved only when a user explicitly approves a proposal; users can edit memories, disable future memory proposals, export their data, and delete their account.

Before public launch, the published version must identify the controller, lawful bases, processors/subprocessors, retention periods, international transfer safeguards, data-subject request path, security measures, children policy, contact details, and change-notice process. Consent records and a regional data-processing inventory must be operated outside this repository.
# Operational telemetry addendum

River records limited operational telemetry needed to keep the service safe and reliable: request route, response status, duration, privacy-hashed abuse-control keys, authentication/security events, aggregate quota use, and voice timing outcomes. It does not place chat text, memory summaries, raw audio, passwords, cookies, API keys, or raw IP addresses in its owner usage dashboard or host request logs. Conversation and approved-memory data remain account-scoped and are handled under the user's selected retention and deletion controls.

The protected owner usage endpoint reports only aggregate counts. It is not an employee activity-monitoring tool and must not be used to profile individual users.
