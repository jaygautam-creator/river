# Contributing to River

Thanks for helping River feel more human, useful, and safe.

## Local workflow

1. Install dependencies with `npm install`.
2. Start the app with `npm run dev`.
3. Before opening a pull request, run `npm run test:smoke`, `npm run build`, and `npm audit --audit-level=high`.

## Product principles

- Memory is opt-in, reviewable, and reversible.
- Never treat River as a crisis service or a substitute for professional care.
- Keep the mobile experience as intentional as desktop.
- Do not introduce secrets, personal data, or production credentials into the repository.

## Pull requests

Describe the user-facing change, include test evidence, and call out privacy or security implications. Keep commits focused and avoid unrelated formatting changes.
